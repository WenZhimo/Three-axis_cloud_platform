# 运行路径术语

以下术语用于区分“代码存在”“参与构建”和“进入当前运行路径”：

- 当前主线：指当前仓库证据能够串起的实际运行流程，可用于描述章节级或系统级数据流。
- 当前 500Hz 实时控制循环：指 `frame_500Hz -> 主循环消费 -> 传感器读取/姿态更新/控制输出` 这一周期路径。
- 当前主控制路径：指 `computeMotorCommands()` 及其下游 `PWM_Motor_SetAngle()`、TIM CCR 写入相关路径。
- 通信支线：指已初始化但不承担当前主控制闭环的通信能力，例如 USB CDC。
- 备用方案：指项目中存在、可作为替代思路分析，但当前没有进入主线的完整方案，例如 `evvgcCF.c` 互补滤波。
- 保留函数：指已经定义但当前没有调用证据的局部函数，例如 `calculateAccConfidence()`。
- 待接入边界：指函数、配置字段或常量看起来有设计意图，但当前没有形成调用或数据流证据，例如 `autoPan()` 与 AutoPan 配置字段。
- 诊断工具：指用于调试或点测的函数，不因存在定义而自动成为当前主线，例如 `PWM_Motor_Hardware_Diagnosis()`。

使用要求：

- 分析总系统时优先使用“当前主线”。
- 分析 `computeMotorCommands()` 到电机输出时使用“当前主控制路径”。
- 分析 USB CDC 等通信能力时使用“通信支线”。
- 分析未接入替代算法时使用“备用方案”，不要写成当前主线。
- 分析未调用的局部辅助函数时使用“保留函数”。
- 分析有设计意图但未形成调用证据的接口或字段时使用“待接入边界”。

# 禁用或需谨慎使用的称呼

以下称呼不得替代正式名称：

- “板子”不得替代 STM32F103RCTx芯片平台，除非上下文明确指硬件整体且有证据。
- “定时器”不得替代通用定时器PWM输出、高级定时器TIM8、基本定时器TIM6。
- “姿态算法”不得替代四元数姿态解算或互补滤波。
- “电机驱动”不得替代三相正弦PWM电机驱动或三轴电机硬件映射。
- “串口”不得替代 UART串口调试输出。
- “USB通信”不得替代 USB FS设备、USB Device中间件或 USB CDC虚拟串口。
