# 穿插节奏总览

第01章：

- 展示项目目标 MCU 和工程边界。
- 不分析控制算法和完整主循环。

第02章：

- 引入工程配置文件、工程目录和构建配置入口。
- 暂不展开源码模块细节。

第03章：

- 引入启动文件、链接脚本和 main 入口关系。
- 只建立复位到应用入口的项目路径。

第04章：

- 引入 HAL 配置、构建产物和实际参与链接的源码集合。
- 建立后续章节的文件索引基础。

第05章：

- 引入 DWT、SysTick、TIM 寄存器访问的项目位置。
- 暂不分析各业务调用。

第06章：

- 引入 SystemClock_Config 和 .ioc 时钟配置。
- 暂不展开具体外设业务。

第07章：

- 引入 stm32f1xx_hal_msp.c、gpio.c 和各外设 MSP 初始化入口。
- 将引脚复用作为后续外设共同前置。

第08章：

- 引入 USART3 重映射和 SWD 保留配置。
- 为后续串口输出和下载调试埋线。

第09章：

- 引入 stm32f1xx_it.c、SysTick_Handler 和周期标志来源。
- 只建立节拍来源，不进入完整控制循环。

第10章：

- 引入 micros 相关位置和 MPU6050 标定中的时间依赖。
- 将 DWT 作为计时资源穿插。

第11章：

- 引入 syscalls.c、sysmem.c、usart.c 和 main.c 中 printf 输出路径。
- 建立后续 10Hz 状态输出的调试支线。

第12章：

- 引入 tim.c 和 drv_pwmMotors.c 中 PWM 输出入口。
- 暂不分析三相电机完整驱动。

第13章：

- 引入 TIM8、TIM6 的初始化和中断配置。
- 标注它们与主业务路径的关系边界。

第14章：

- 引入 i2c.c 和 mpu6050.c 的 I2C 访问路径。
- 为 MPU6050 驱动章节做项目入口。

第15章：

- 引入 USB_DEVICE 和 Middlewares 中间件目录。
- 标注 USB 属于已初始化通信支线。

第16章：

- 引入 usbd_cdc_if.c、usbd_desc.c、usbd_conf.c。
- 明确当前未发现上位机业务协议解析。

第17章：

- 引入 mpu6050.h、bgc32.h、config.c、drv_pwmMotors.h。
- 建立后续传感器、控制和电机模块的数据结构索引。

第18章：

- 引入 MPU6050_Init 和寄存器配置相关文件。
- 只处理初始化入口，不进入完整数据链。

第19章：

- 引入 MPU6050_Read_And_Process 到 main 数据入口。
- 将原始数据读取作为传感器主线开始。

第20章：

- 引入 orientIMU 和安装方向矩阵相关位置。
- 将坐标处理放在姿态输入之前。

第21章：

- 引入 mpu6050Calibration.c 和运行时零偏相关位置。
- 将标定、零偏和物理量缩放作为姿态估计前的数据质量环节。

第22章：

- 引入 firstOrderFilter.c 和 MargAHRS.c 中的重力校正位置。
- 只定位滤波入口，不展开算法正文。

第23章：

- 引入 MargAHRSupdate、欧拉角输出和角度归一化使用位置。
- 首次形成姿态输出到控制输入的主线桥接。

第24章：

- 引入 evvgcCF.c 作为备用姿态方案。
- 标注其与主循环当前调用方案的关系。

第25章：

- 引入 sensors_t 和 sensors.margAttitude500Hz。
- 将状态结构作为 PID 输入容器。

第26章：

- 引入 frame_500Hz、main 循环、micros 和周期任务入口。
- 形成实时控制循环骨架。

第27章：

- 引入 pid.c 和 computeMotorCommands.c。
- 首次进入闭环控制核心。

第28章：

- 引入 D 项滤波、目标角斜坡、输出限幅、积分状态相关模块。
- 将控制细节作为安全输出前置。

第29章：

- 引入机械角、电角和 fastTrig.c。
- 将 PID 输出过渡到电机相位输出。

第30章：

- 引入 PWM_Motor_SetAngle、三相 PWM 和 Roll/Pitch/Yaw 通道映射。
- 首次形成控制输出到三轴电机的执行链。

第31章：

- 引入 main.c 与 computeMotorCommands.c 的系统级组合关系。
- 汇总传感器、姿态、PID、电机输出四条主线。

第32章：

- 引入 10Hz 输出、运行状态门控和异常值防护。
- 建立项目稳定性和运行状态边界。

第33章：

- 引入 Debug.launch、SWD 配置和电机硬件诊断入口。
- 将项目从代码理解推进到硬件调试路径。

第34章：

- 引入 tools/pid_tuning_sim.py。
- 将调参、移植、扩展和类似系统设计作为最后收束。

# 项目内容引入原则

- 第01至第09章只建立工程、启动、时钟、中断和 GPIO 基础。
- 第10至第16章只引入项目已使用或已初始化的外设支线。
- 第17至第21章进入传感器驱动和数据结构。
- 第22至第28章进入姿态估计和控制算法。
- 第29至第32章整合执行器和系统运行结构。
- 第33至第34章进入调试、诊断、调参和迁移扩展路线。
