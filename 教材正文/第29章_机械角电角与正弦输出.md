# 第29章 机械角、电角与正弦输出

## 1. 本章目标

第28章已经把 PID 输出限制在较安全的角度补偿范围内。本章继续回答一个更靠近执行器的问题：这些角度量如何进入电机相位输出。

本章目标是：

- 理解 `机械角与电角转换` 为什么是云台姿态角和电机定子相位之间的桥梁。
- 理解机械角、电角、定子电角和 PID 补偿角之间的区别。
- 理解项目如何用 `mechanical2electricalDegrees[]` 和 `electrical2mechanicalDegrees[]` 表示换算关系。
- 理解 `正弦查表快速三角函数` 的基本思想，以及项目中两套正弦表的边界。
- 能从 `computeMotorCommands()` 追踪到 `PWM_Motor_SetAngle()` 的角度输入，但不把本章误写成完整三相 PWM 硬件映射。

本章知识链为：

-> `知识依赖图` 中它们依赖 `角度归一化`、`枚举定义电机轴` 或作为查表起点

## 3. 问题背景

三轴云台的姿态角来自 IMU 和姿态解算，表示的是云台结构在空间中的机械姿态。例如 Pitch 轴抬起多少弧度，Yaw 轴转过多少弧度。

电机底层需要的不是“云台机械角应该是多少”，而是“定子三相磁场应该放在哪个电角位置”。如果直接把机械角当成电机相位角，电机磁场与转子位置就会错位，输出方向、输出幅值和闭环补偿都可能异常。

因此本项目的执行链路中间需要一层转换：

```text
姿态机械角
-> PID补偿角
-> 定子电角
-> 正弦表索引
-> 三相CCR目标值
```

第27章和第28章解决的是 PID 输出如何得到、如何限制。本章解决的是 PID 输出进入电角和正弦表之前的含义转换。

## 4. 核心概念

### 4.1 机械角

机械角描述真实机构转过的角度。项目中 `sensors.margAttitude500Hz[ROLL]`、`sensors.margAttitude500Hz[PITCH]`、`sensors.margAttitude500Hz[YAW]` 属于姿态角来源，它们先经过符号、归一化、误差计算和 PID，再参与电机输出。

机械角的单位在项目控制链中主要是弧度。

### 4.2 电角

电角描述电机定子磁场相位。对于多极电机，转子机械转一圈，电角会转多圈。一般关系可以写成：

```text
电角 = 机械角 * 极对数
```

项目变量名使用 `mechanical2electricalDegrees[]`，但当前代码中的角度计算主要以弧度参与运算。因此阅读时不要被 `Degrees` 命名误导，本章按“机械角到电角的换算系数”理解它。

### 4.3 定子电角

定子电角是最终传给 `PWM_Motor_SetAngle()` 的角度参数。它不只是传感器当前角度，也可能叠加 PID 补偿、符号方向和固定偏移。

例如 Roll 后续 PID 分支中：

```text
stator_electrical_angle = current_electrical_angle + ROLL_STATOR_SIGN * pidCmd[ROLL]
```

这里 `pidCmd[ROLL]` 是电角方向上的补偿量，不是 PWM 占空比。

### 4.4 正弦查表快速三角函数

正弦查表是把一圈正弦值预先存入数组，运行时用角度换算成索引，再从数组中取近似正弦值。这样可以减少实时计算 `sinf()` 的开销。

当前项目中有两套相关实现：

- `Drivers/SRC/Src/fastTrig.c` 定义 `sinDataI16[SINARRAYSIZE]`，由 `initSinArray()` 生成 1024 点正弦表，`fastSin()` 用索引近似正弦。
- `Drivers/CustomDrivers/Src/drv_pwmMotors.c` 内部定义 `sinTable[SIN_TABLE_SIZE]`，由 `GenerateSinTable()` 生成 720 点正弦表，`PWM_Motor_SetAngle()` 当前实际使用这张表生成三相 CCR 目标值。

因此，`fastTrig.c` 是项目中的查表示例和工具模块；当前主电机输出链路的正弦表证据在 `drv_pwmMotors.c`。

## 5. 工作原理

### 5.1 从机械误差到电角误差

以 Roll 后续 PID 分支为例，项目先得到当前机械角：

```text
roll_angle = wrapToPif(ROLL_SENSOR_SIGN * sensors.margAttitude500Hz[ROLL])
```

再计算目标与当前的机械角误差：

```text
roll_error_mech = wrapToPif(rollTargetSlew - roll_angle)
```

随后进入机械角到电角的转换：

```text
current_electrical_angle = roll_angle * mechanical2electricalDegrees[ROLL]
target_electrical_angle = current_electrical_angle + roll_error_mech * mechanical2electricalDegrees[ROLL]
```

这一步的意义是：姿态误差发生在机械结构上，但电机相位补偿必须按电角尺度计算。

### 5.2 PID输出不是PWM占空比

`updatePID()` 返回后，项目把结果放入 `pidCmd[]`，再经过第28章讲过的限幅和速率限制。这个 `pidCmd[]` 仍然是角度补偿量。

Roll 分支中，`pidCmd[ROLL]` 加到当前电角上：

```text
current_electrical_angle + ROLL_STATOR_SIGN * pidCmd[ROLL]
```

Pitch 分支中，`pidCmd[PITCH]` 加到 `current_elec` 和 `pitch_elec_offset` 上：

```text
current_elec + pitch_elec_offset + PITCH_STATOR_SIGN * pidCmd[PITCH]
```

Yaw 分支中，`pidCmd[YAW]` 与当前电角组合：

```text
-current_elec + YAW_STATOR_SIGN * pidCmd[YAW]
```

所以 PID 输出不是直接写入 TIM 的 CCR，也不是直接等于占空比。它先决定定子电角，再由 `PWM_Motor_SetAngle()` 转成正弦三相输出。

### 5.3 电角归一化

电角可以不断增加或出现负值，但正弦表只需要一圈内的位置。

`computeMotorCommands.c` 会在部分分支用 `wrapToPif()` 把角度限制在 `[-PI, PI]`。`PWM_Motor_SetAngle()` 内部又使用：

```text
normalized_angle = fmodf(angle_rad, 2π)
如果 normalized_angle < 0，则加 2π
```

这说明上层可以传入正负电角，底层会把它归一化到 `[0, 2π)` 后再查表。

### 5.4 从电角到正弦表索引

`PWM_Motor_SetAngle()` 中的索引计算为：

```text
index = normalized_angle * SIN_TABLE_SIZE / (2π)
```

`SIN_TABLE_SIZE` 在 `drv_pwmMotors.h` 中为 720。也就是说，一圈电角被分成 720 个采样点。角度越大，索引越靠后；角度绕回一圈后，索引重新回到 0 附近。

### 5.5 三相相差120度

底层正弦输出需要 A/B/C 三相相差 120 度。`PWM_Motor_SetAngle()` 当前用表索引偏移实现：

```text
sin_a = sinTable[index]
sin_b = sinTable[(index + SIN_TABLE_SIZE / 3) % SIN_TABLE_SIZE]
sin_c = sinTable[(index + 2 * SIN_TABLE_SIZE / 3) % SIN_TABLE_SIZE]
```

本章只讲到“三相目标值来自同一电角的三个相位偏移”。至于这些值具体写入 TIM2/TIM3/TIM4 哪些 CCR 通道，属于第30章。

## 6. STM32实现机制

本章涉及 STM32 的方式有三层。

第一，`Core/Src/main.c` 先调用 `MX_TIM3_Init()`、`MX_TIM2_Init()`、`MX_TIM4_Init()`，再调用 `PWM_Motor_Init()`。这说明 PWM 外设已经初始化后，电机驱动模块才生成正弦表并启动 PWM 通道。

第二，`PWM_Motor_Init()` 调用 `GenerateSinTable()`，并启动 TIM2、TIM3、TIM4 的 PWM 通道。它还把 TIM2、TIM3、TIM4 的计数器清零，用于减小跨定时器相位错位。完整相位同步属于第30章，本章只保留这个入口关系。

第三，`PWM_Motor_SetAngle()` 根据电角和功率百分比计算 CCR 目标值。`Core/Src/tim.c` 中 TIM2、TIM3、TIM4 当前 `Period` 都是 3599，`PWM_Motor_SetAngle()` 使用 `htim3.Init.Period + 1` 作为周期基准，并假设三个定时器 ARR 一致。这个假设当前有 `tim.c` 配置证据。

## 7. 项目中的应用

本章涉及的项目证据文件包括：

- `Drivers/SRC/Src/computeMotorCommands.c`
- `Drivers/SRC/Inc/computeMotorCommands.h`
- `Drivers/SRC/Src/fastTrig.c`
- `Drivers/SRC/Inc/fastTrig.h`
- `Drivers/CustomDrivers/Src/drv_pwmMotors.c`
- `Drivers/CustomDrivers/Inc/drv_pwmMotors.h`
- `Drivers/SRC/Src/config.c`
- `Core/Src/main.c`
- `Core/Src/tim.c`

项目中的运行关系是：

```text
Core/Src/main.c
-> PWM_Motor_Init()
-> computeMotorCommands()
-> PWM_Motor_SetAngle()
-> sinTable查表
-> CCR目标值
```

`computeMotorCommands.c` 负责把姿态角、目标角和 PID 补偿组合为定子电角。

`drv_pwmMotors.c` 负责把定子电角转换成正弦三相目标值。

`fastTrig.c` 提供独立的 `fastSin()` 查表函数，但当前搜索结果只发现定义，没有发现主控制输出路径调用它。因此本章把它作为 `正弦查表快速三角函数` 的项目实现证据之一，而不把它写成当前电机输出主路径。

`config.c` 设置 `rollMotorPoles`、`pitchMotorPoles`、`yawMotorPoles`，但当前没有发现 `computeMotorCommands.c` 读取这些字段参与换算；`drv_pwmMotors.c` 中的 `POLE_PAIRS` 当前也没有发现主输出链路使用证据。因此这些配置和常量只能作为存在证据，不能写成当前运行时换算来源。

## 8. 代码分析

### 8.1 换算系数数组

`computeMotorCommands.c` 定义：

```text
mechanical2electricalDegrees[3] = {7.0f, 7.0f, 7.0f}
electrical2mechanicalDegrees[3] = {1/7, 1/7, 1/7}
```

这两个数组表示机械角和电角之间的换算关系。`computeMotorCommands.h` 用 `extern` 暴露它们，说明它们是可被其他模块观察或使用的控制状态。

需要注意的是，Roll 后续 PID 分支会把 `mechanical2electricalDegrees[ROLL]` 改为 `14.0f`，Pitch 在 `return_state == false` 分支会把 `mechanical2electricalDegrees[PITCH]` 改为 `20.0f`。Yaw 当前使用数组已有值，未看到同类分支内重写证据。

因此三轴当前换算系数不是完全统一的配置读取结果，而是包含初值和分支内写入。

### 8.2 Roll输出路径

Roll 有两个路径。

回中路径中，代码把 `rollTargetSlew` 乘以 `mechanical2electricalDegrees[ROLL]`，叠加 `ROLL_STATOR_SIGN` 和 `ROLL_ELEC_OFFSET` 后，传给 `PWM_Motor_SetAngle()`。

后续 PID 路径中，代码先计算 `current_electrical_angle` 和 `target_electrical_angle`，PID 输出限幅后，再生成：

```text
stator_electrical_angle = current_electrical_angle + ROLL_STATOR_SIGN * pidCmd[ROLL]
```

这条路径清楚体现了“机械角误差 -> 电角目标 -> PID电角补偿 -> 定子电角”。

### 8.3 Pitch输出路径

Pitch 先对传感器角乘 `PITCH_SENSOR_SIGN`，再用 `pointingCmd[PITCH]` 和 `pitch_angle` 执行 PID。随后代码根据 `return_state` 选择电角计算方向：

```text
return_state == true:  current_elec = pitch_angle * mechanical2electricalDegrees[PITCH]
return_state == false: current_elec = -pitch_angle * mechanical2electricalDegrees[PITCH]
```

最后叠加：

```text
pitch_elec_offset + PITCH_STATOR_SIGN * pidCmd[PITCH]
```

其中 `pitch_elec_offset = -1.25f` 是固定偏移。该偏移有源码证据，但缺少更高层说明来源，本章只按项目常量记录，不推测机械安装原因。

### 8.4 Yaw输出路径

Yaw 先用 `wrapToPif()` 得到机械误差，再计算 `safe_target` 进入 PID。输出阶段：

```text
current_elec = yaw_angle * mechanical2electricalDegrees[YAW]
stator_electrical_angle = -current_elec + YAW_STATOR_SIGN * pidCmd[YAW]
```

Yaw 的当前电角前面带负号，说明该轴输出方向和传感器/电机安装方向之间存在符号约定。源码有这个符号证据，但本章不进一步猜测硬件安装原因。

### 8.5 fastTrig.c的查表函数

`fastTrig.c` 的流程是：

```text
initSinArray()
-> 生成 1024 点 sinDataI16[]
-> fastSin(x) 把 x 映射为索引
-> 返回缩放后的浮点正弦近似值
```

`fastSin()` 对负角使用 `-x` 查表再取负值。它说明了查表法的基本思想：把实时三角函数计算换成数组访问和缩放。

当前没有发现 `initSinArray()` 或 `fastSin()` 在主控制输出路径中被调用。因此调试电机输出时，不应只看 `fastTrig.c`，还必须看 `drv_pwmMotors.c`。

### 8.6 drv_pwmMotors.c的输出查表

`drv_pwmMotors.c` 的 `GenerateSinTable()` 生成 720 点 `sinTable[]`。`PWM_Motor_SetAngle()` 收到电角后：

1. 把电角归一化到 `[0, 2π)`。
2. 把电角映射为 `sinTable` 索引。
3. 取 A/B/C 三相相差 120 度的正弦值。
4. 按 `power_percent` 缩放幅值。
5. 加上 `half_period` 得到 CCR 目标值。

这就是本章“正弦输出”的核心。CCR 写入哪些定时器通道，留到第30章展开。

## 9. 调试方法

第一步，观察机械角到电角换算。

- Roll 观察 `rollDiag.currentCtrlMechRad`、`rollDiag.currentElecRad`、`rollDiag.targetElecRad` 和 `mechanical2electricalDegrees[ROLL]`。
- Pitch 观察 `pitch_angle`、`current_elec` 和 `mechanical2electricalDegrees[PITCH]`。
- Yaw 观察 `yaw_angle`、`current_elec` 和 `mechanical2electricalDegrees[YAW]`。

第二步，确认 PID 输出仍是角度补偿。

- 观察 `pidCmd[]` 和 `stator_electrical_angle` 的关系。
- 如果 `pidCmd[]` 变化但 CCR 没有按预期变化，应继续进入 `PWM_Motor_SetAngle()`，而不是把 `pidCmd[]` 当成占空比。

第三步，观察正弦表初始化。

- 在 `PWM_Motor_Init()` 后确认 `isInitialized` 为 1。
- 确认 `GenerateSinTable()` 已在初始化时执行。
- `fastTrig.c` 的 `sinDataI16[]` 只有在 `initSinArray()` 被调用后才有有效表值；当前主输出链路未见调用证据。

第四步，观察电角归一化和索引。

- 在 `PWM_Motor_SetAngle()` 中观察 `angle_rad`、`normalized_angle` 和 `index`。
- 当输入角为负时，确认 `normalized_angle` 被转换到 `[0, 2π)`。

第五步，观察正弦三相目标值。

- 观察 `sin_a`、`sin_b`、`sin_c` 是否来自相差 `SIN_TABLE_SIZE / 3` 的索引。
- 观察 `ccr_val[]` 是否围绕 `half_period` 上下变化。

这些调试方法只定位到“电角和正弦输出目标值”。第30章再继续观察 TIM2/TIM3/TIM4 的具体 CCR 通道映射。

## 10. 常见问题

问题一：`mechanical2electricalDegrees[]` 名字里有 Degrees，是否表示角度单位是度。

不应这样理解。当前项目控制链中多数角度以弧度计算，`mechanical2electricalDegrees[]` 在代码中实际作为机械角到电角的倍数使用。变量名保留源码原名，教材解释按运行行为。

问题二：PID 输出是否直接控制 PWM 占空比。

不是。PID 输出进入 `pidCmd[]`，再参与 `stator_electrical_angle` 计算。占空比目标值由 `PWM_Motor_SetAngle()` 根据电角、正弦表和功率百分比计算。

问题三：`config.c` 中的 `rollMotorPoles`、`pitchMotorPoles`、`yawMotorPoles` 是否驱动当前换算。

当前没有发现 `computeMotorCommands.c` 读取这些字段参与换算的证据。它们是配置结构中的字段和默认值，但当前换算来源主要是 `mechanical2electricalDegrees[]` 的初值和分支内赋值。

问题四：`fastSin()` 是否就是当前电机输出的正弦函数。

当前证据不是。`fastSin()` 存在于 `fastTrig.c`，但主电机输出路径中的正弦表是 `drv_pwmMotors.c` 的内部 `sinTable[]`。

问题五：为什么电角需要归一化。

正弦表只表示一圈。电角可能为负，也可能超过一圈。归一化后才能稳定映射到 `[0, SIN_TABLE_SIZE)` 的有效索引范围。

问题六：为什么三相索引相差 `SIN_TABLE_SIZE / 3`。

一圈分成三等份就是 120 度。三相正弦输出需要 A/B/C 三相相位相差 120 度，因此用三分之一表长作为索引偏移。

7. 如果后续要让 `config.c` 中的电机极对数字段真正参与换算，应优先检查哪些调用关系。

## 13. 本章总结

本章把第28章的 PID 输出继续连接到电机相位输出前端。

当前项目中：

- `mechanical2electricalDegrees[]` 和 `electrical2mechanicalDegrees[]` 表示机械角与电角换算关系。
- Roll、Pitch、Yaw 的电角计算路径并不完全相同，存在不同的符号、偏移和分支写入。
- `pidCmd[]` 是角度补偿量，不是 PWM 占空比。
- `PWM_Motor_SetAngle()` 的输入 `angle_rad` 是电气角度。
- `PWM_Motor_SetAngle()` 内部把电角归一化到 `[0, 2π)`，再映射到 `SIN_TABLE_SIZE=720` 的正弦表索引。
- `fastTrig.c` 提供 1024 点 `fastSin()` 查表实现，但当前主电机输出链路使用 `drv_pwmMotors.c` 内部 720 点 `sinTable[]`。
- `config.c` 中存在三轴电机极对数字段，但当前未发现它们参与 `computeMotorCommands.c` 的换算。

本章边界是：只讲机械角、电角、正弦查表和 `PWM_Motor_SetAngle()` 的角度输入逻辑；不展开 TIM2/TIM3/TIM4 具体通道写入和三轴硬件映射。

下一章将进入 `多定时器三相电机驱动`，继续说明 `PWM_Motor_SetAngle()` 如何把三相 CCR 目标值写入不同定时器通道，并形成 Roll、Pitch、Yaw 的实际执行链。
